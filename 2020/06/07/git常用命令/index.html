git常用命令

  • 2020-06-07

  修改代码时，为了防止修改前的代码以后还用的上，往往每次都要建立文件夹来存放以前的代码，不知不觉建立了10多个文件夹来存放同一个项目的不同版本的代码，感觉特别的臃肿。所以决定拿出点时间学习下git，方便以后文件给管理。磨刀不误砍柴工嘛！

git简介

  git是分布式版本管理系统，所谓分布式是相对于SVN这种集中式版本管理系统来说的，git的使用不需要集中存放版本的中央服务器，每个人的电脑都是一个版本库，所以叫分布式。而版本管理顾名思义就是管理文件的不同版本，你可以取出你任意时间修改的文件的版本，而不用像以前一样自己建立文件夹去存放它们。

常用的git命令

  感觉一门知识的学习，在初期接触的时候，会给人一种清楚简洁的印象，但随着接触的深入，会发现并没有那么简单，坑也不少。所以这里也不求完全理解，先列出来一些简单常用的git命令，用起来再说。

  • 首先新建文件夹 ，在文件夹中Git Bash here,打开命令窗口后输入git init进行初始化，这样我们就成功建立了一个仓库用来存放我们日后各个文件的版本。
  • 建立好之后我们就可以存放提交我们的文件，比如新建readme.txt文件，使用命令git add readme.txt将文件添加到暂缓区，接着使用git commit -m "wrote a readme file"命令将文件提交到仓库，-m后是对本次提交的说明，这个建议写清楚一些，以方便日后对版本的查找。
  • 假设我们对readme.txt进行多次修改并提交到仓库中，我们突然想查看前几次的版本，首先利用命令git reflog(为使输出简洁可以使用git reflog --pretty=oneline)来查看每次的提交日志和commit id,知道commit id后，再利用git reset --hard commit id来恢复到我们想要的版本。注意commit id写几位就行，不用写全。如果知道要回退第几个版本也可以用命令git reset --hard HEAD~n。
  • 要撤销工作区文件的修改，使用命令git checkout -- file要撤销添加到暂存区的文件的修改，在使用上一个命令之前，先使用命令git reset HEAD file,若提交到版本库那就版本回退。
  • 若要删除文件，先在文件夹中删除文件，然后运行命令git rm filename，然后用commit命令提交即可。
  • 直接创建分支git branch name,要查看分支使用git branch分支创建并切换，使用switch命令，如创建dev分支git switch -c dev即创建并切换到新的dev分支，直接切换到已有的master分支，则是git switch master。删除分支为git branch -d name
  • 对于分支的合并，有多种方式，这里我们使用常用的git merge --no-ff -m "说明" branchname命令。再利用git log --graph --pretty=oneline --abbrev-commit可以查看分支历史。
